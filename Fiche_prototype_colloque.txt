Recherches :

* Piste personnelle (slide 2) :

- Recherches Threshold, substitut à DAOStarFinder, seuils etc... via la documentation OpenCV 
Threshold -> set un seuil entre 0 et 255, pour chaque pixel : analyse si sa valeur est supérieure ou inférieure au seuil. Inférieure : pixel = 0 et supérieure = 255. Pas la meilleure utilisation dans notre cas… Mais : Adaptative Threshold intéressant. 
Utilisation de l'adaptative Threshold, analyse les pixels AUTOUR du pixel en question, ce qui permet une meilleure précision et d'atténuer l'éclat des étoiles. On fixe le rayon qui vérifie les pixels autour du pixel à l'aide du noyau Gaussien.

- Recherches Opening, opening équivaut à un dilating après un eroding -> opening(…) <=> dilating(eroding(…)). Opening permet d'obtenir une image starless bien plus propre et sans étoile apparente et nette. 
"Useful for removing small objects (it is assumed that the objects are bright on a dark foreground)" = utile pour retirer des petits objets (implique que les objets soient clairs dans un fond sombre) -> exactement la situation des étoiles trop brillantes dans une photo.

- Recherches Astrometry.net, utile pour faire un masque d'étoile bon car utilise un algorithme et une base de donnée pour identifier les étoiles présentes dans une photo. Permet donc de les isoler précisément et donc d'obtenir un masque proche de la réalité.
s'utilise avec Linux (ou Windows WSL) et c'est une API.
